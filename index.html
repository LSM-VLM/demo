Abstract

Recent vision-language models show strong perceptual and logical abilities, but still struggle with 3D spatial question answering when evidence is distributed across long trajectories or hidden behind unvisited viewpoints. LSM-VLM addresses this by using structured cognitive memory as the interface between perception, imagination, and reasoning. From an input video, it builds a dual memory: a scene graph for object-centric long-term semantics and a local BEV map for short-term geometric grounding. Conditioned on the question and current memory, the model predicts informative target viewpoints, queries a world model to imagine missing observations, converts the imagined rollout into candidate graph and BEV updates, and selectively consolidates them through confidence-aware recurrent memory updates. This design improves long-horizon and cross-view spatial reasoning on VSI-Bench and SQA3D.

LSM-VLM alternates between a world-model flow for collecting missing spatial evidence and a VLM answer flow for reading updated memory and deciding whether to answer.

Overview

The updated paper frames the system as two alternating closed loops: a World Model Flow that plans, imagines, detects, and updates memory, and a VLM Answer Flow that reads original observations, imagined evidence, and dual memory to produce the final answer.

Method Components

Long-Term Scene Graph

Object nodes store category, 2D world-frame center, oriented extent, confidence, and NYU40 semantic encoding; graph edges store 3DSSG-style spatial relations.

Short-Term BEV Map

A local agent-centered grid keeps occupancy and semantic channels, preserving direction-sensitive geometry while avoiding global BEV redundancy.

Recurrent Memory Update

BEV memory forgets low-confidence cells and injects transformed candidate updates; graph updates are committed only after verification against consolidated BEV evidence.

Structured Spatial Evidence

The new version makes the memory definitions more concrete. The BEV map is a two-channel local grid for occupancy and NYU40 semantic labels, while the scene graph is a structured JSON memory with object nodes, agent state, and spatial relations.

VGGT and SAM3 are used for perceptual grounding and for parsing imagined rollouts, keeping the real and imagined observations aligned through the same perception module.

Training Recipe

  • Backbone: Qwen3-VL-8B-Instruct, with Qwen3-VL 4B and 8B variants used in training ablations.
  • World model: Stable Virtual Camera generates target-pose rollouts.
  • Stage 1: train map and graph encoders plus projectors while freezing the original VLM modules.
  • Stage 2: continue task adaptation with LoRA while keeping graph and map pathways trainable.
  • Data: indoor embodied-scene training splits with two-stage spatial-memory adaptation.

Ablation Takeaways

  • Adding only BEV or only graph gives little gain; the dual memory is useful because geometric and semantic memories complement each other.
  • World-model imagination alone helps modestly, but its gain becomes much larger when imagined views are converted into structured memory updates.
  • Recursive memory update raises VSI-Bench from 67.1 to 69.2 and especially improves relative distance, relative direction, and route planning.
  • The best asynchronous memory configuration updates BEV every round and graph every four rounds, matching their short-term and long-term roles.

Efficiency

The updated paper reports an average inference time of about 67.0 seconds per sample. VLM calls contribute 19.68 seconds in total, the world model contributes 13.80 seconds, VGGT and SAM3 each contribute 15.26 seconds across dense frame calls, and memory update itself remains lightweight at 2.97 seconds.

Limitations

Current experiments focus on Qwen3-VL 4B and 8B backbones. Future work should evaluate transfer to broader VLM families, parallelize VLM and world-model execution to reduce latency, and condition the world model directly on structured memory to improve the consistency and task relevance of imagined views.
